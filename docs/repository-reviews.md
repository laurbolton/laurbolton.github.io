## Repository 1

**Repository Name:** QGIS  
**Link:** https://github.com/qgis/QGIS  

### Purpose
QGIS is an open source geographical information system designed to work across platforms.

### Repository Structure
A couple folders that stand out to me are .ci and .docker which both provide test files for users to run pre-commit. I also find it interesting that there are folders that were updated as long as 21 years ago and as recently as 15 minutes ago. 

### Documentation
The README is extremely helpful. It provides real-world visual examples of the cartographic, analytical, and customization features available for users. It also provides other resources including installation guides, documentation, and contribution guidelines. 

### Activity
Yes, the repository is actively maintained. There are around 100 commits per week and over 96,000 commits total.

### Key Observation
Considering how large this repository is, this project has a very effective organizational structure. It provides clear community guidelines which makes it possible for the project to have many contributors.

## Repository 2

**Repository Name:** GDAL  
**Link:** https://github.com/OSGeo/gdal  

### Purpose
GDAL is a translation library for both raster and vector data formats. It is an open source platform that provides the ability to read, write, and modify data.

### Repository Structure
The “NEWS” file stands out to me - it gives a detailed overview of all the changes that developers made for the new release (GDAL 3.13.0). All changes are documented in the repository, but it is helpful to see them all in one single file. 

### Documentation
The README provides a contribution guide, code of conduct, bug tracking, and other helpful resources. It is missing some examples for how GDAL is used, but it links to the main website where users can learn more. 

### Activity
Yes, the repository is actively maintained. There are around 50-100 commits per week and over 63,000 commits total.

### Key Observation
This project has an effective way to track bugs with issue templates and streamlined workflows.

## Repository 3

**Repository Name:** Cesium  
**Link:** https://github.com/CesiumGS/cesium  

### Purpose
Cesium is a Javascript library that provides the functionality to develop 3D maps. It is free, open-source, and can be used in a web browser.

### Repository Structure
The Contributors file stood out to me, because it lists all of the corporations and individuals that have contributed to the project. I also like that some folders (Documentation) have their own individual README file as a landing page for the content inside that folder.

### Documentation
The README provides a helpful overview of the project and external links to the website, real-world examples, and user stories. There are also other tabs for the code of conduct and contribution guide.

### Activity
Yes, the repository is actively maintained. There are between 25-100 commits per week and almost 45,000 commits total.

### Key Observation
This project’s Wiki page stands out, because it provides one centralized location for all user guides and documentation.

## Repository 4

**Repository Name:** Mesa  
**Link:** https://github.com/mesa/mesa  

### Purpose
Mesa is a Python library that enables users to build agent-based models in a browser-based interface.  

### Repository Structure
Mesa is smaller than the other projects that I’ve reviewed, but the repository is very organized. One file that stands out is the version history (HISTORY.md), which provides detailed updates for every release that occurs.  

### Documentation
The README is very helpful. It provides real-world visual examples, links to demos, and other external resources for users. It also provides a code of conduct and contribution guide similar to the other projects. 

### Activity
Yes, the repository is actively maintained. There are between 5-20 commits per week and 3,000 commits total.

### Key Observation
This project is smaller, but it has a very effective organizational structure. Individual folders have their own README files, which makes it easy for the user to find/use the files within.

## Reflection

Repositories are a collection of code and files that track changes to the project over time. Some of the most important components are the README, license, .gitignore, and organized directories. Repositories foster collaboration by providing a way for users to document issues with the project functionality. Individuals can collaborate on open source projects by finding solutions to these outstanding issues and committing changes.
